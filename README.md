# 📋 Облік контролів та протокольних рішень

Веб-застосунок для обліку та контролю виконання наказів, заходів та протокольних рішень з можливістю зберігання даних у браузері (IndexedDB).

## 🚀 Особливості

- **Локальне зберігання даних** — використання Dexie.js (обгортка над IndexedDB) для зберігання даних у браузері
- **Повноцінне CRUD** — додавання, редагування, видалення записів
- **Фільтрація** — пошук за назвою, номером наказу, відповідальним; фільтр за статусом
- **Статистика** — візуалізація кількості записів за статусами та напрямками
- **Прикріплення PDF** — можливість додавати PDF-файли до записів (зберігаються у Base64)
- **Експорт/імпорт** — вивантаження даних у JSON та Excel (.xlsx) формати
- **Адаптивний дизайн** — зручне відображення на різних пристроях

## 📦 Технології

- **HTML5 / CSS3** — структура та стилі
- **JavaScript (ES6+)** — логіка застосунку
- **Dexie.js** — робота з IndexedDB
- **SheetJS (XLSX)** — експорт у Excel
- **FileSaver.js** — збереження файлів

## 🛠️ Встановлення та запуск

1. **Клонуйте репозиторій:**
   ```bash
   git clone https://github.com/your-username/controls-management.git
   cd controls-management
2. **Запустіть файл:**
   index.html штатни браузером.

   📖 Використання

Додавання запису

    Заповніть форму:
        Назва та номер наказу
        Дата наказу
        Заходи для виконання
        Періодичність виконання
        Термін виконання
        Відповідальний
        Напрямок (при виборі "Інше" з'явиться додаткове поле)
        Статус
    За бажанням додайте PDF-файл
    Натисніть "Додати запис"

Редагування
    Натисніть кнопку ✏️ у рядку таблиці
    Відредагуйте дані у формі
    Натисніть "Зберегти" (кнопка змінюється на час редагування)

Видалення
    Натисніть кнопку 🗑️ у рядку таблиці
    Підтвердіть видалення

Фільтрація
    Використовуйте поле 🔍 Пошук для пошуку за текстом
    Оберіть статус у випадаючому списку для фільтрації

Експорт даних
    Excel — вивантаження поточного відфільтрованого списку у .xlsx формат
    JSON — доступний через імпорт/експорт (функціонал у коді)

📊 Структура даних

Кожен запис містить:
Поле	                    Тип	Опис
id	           number	   Автоінкрементний ідентифікатор
orderName	     string           	Назва наказу
orderNumber    string	            Номер наказу
orderDate	     string	            Дата наказу (YYYY-MM-DD)
measures	     string	            Заходи для виконання
periodicity	   string	            Періодичність (Одноразово, Щодня, тощо)
deadline	     string	            Термін виконання (YYYY-MM-DD)
responsible    string	            Відповідальна особа
direction	     string	            Напрямок діяльності
status	       string	            pending / in-progress / completed
pdfData        string (Base64)	  Дані PDF-файлу (опціонально)
pdfName        string	            Назва PDF-файлу
